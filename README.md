## Features

| Feature | Description |
|---|---|
| **Snippet Management** | Create, edit, delete, and search Knowledge Transfer snippets with titles, categories, tags, and rich body text. |
| **Account Tagging** | Organise snippets by customer or project account; filter the entire vault by a single account at a time. |
| **AI Extraction** | Upload raw text files (Teams summaries, incident logs, runbooks) and have LMStudio extract structured snippets automatically. |
| **Grammar & Content Review** | One-click AI review that flags spelling/grammar issues, missing steps, unclear references, security gaps, and best-practice concerns. |
| **Enhancement Suggestions** | Actionable improvement suggestions generated by the same review pipeline, displayed in a dedicated tab. |
| **Markdown Reports** | Generate a per-account Markdown report (category-grouped, tagged) with one click â€“ great for handover documentation or audit trails. |
| **JSON Import / Export** | Full backup and mobility via JSON export; merge-friendly import that deduplicates by title + category. |
| **Responsive Dark UI** | Tailwind CSS dark-mode dashboard, works on desktop and tablet. |

### Data Model

- **Account** â€“ A customer or project. Holds a name, optional description, and timestamp. Snippets cannot be deleted while linked to an account; the account must first be cleared.
- **Snippet** â€“ One piece of knowledge transfer. Linked to exactly one Account. Fields: `title`, `category`, `body`, `tags` (comma-separated), `timestamp`.

## Running the Application

```bash
python app.py
```

The server starts on `http://127.0.0.1:5000`. To change the port or host, edit `app.run(...)` in `app.py` (last line).

### Environment Variables

| Variable | Default | Purpose |
|---|---|---|
| `SECRET_KEY` | `kt-vault-dev-key` | Flask session secret; override for production |
| `LMSTUDIO_ENDPOINT` | `http://127.0.0.1:1234` | URL of the LMStudio server (OpenAI-compatible) |
| `LMSTUDIO_MODEL` | `ornith-1.0-35b` | Model name sent to LMStudio |

## Using KT Assistant

### 1. Create an Account
Click **Accounts** in the header toolbar â†’ **+ Add Account** â†’ enter a customer/project name (required) and optional description â†’ Save.

### 2. Capture Snippets
Switch to the **New Snippet** tab, select an account from the dropdown, fill in title / category / tags / body, then click **Save Snippet**. AI review is optional â€“ use **Review & Enhance** for grammar + content analysis powered by LMStudio.

### 3. Extract from a Text File
Click **Load File**, choose a `.txt`, `.md`, or `.log` file, and the AI will extract structured snippets automatically. Review the extracted list in the panel, pick an account to save them under, then click **Save All**.

### 4. Search & Filter
Use the search bar (top-right) for real-time filtering across title, category, body, tags, and account name. Switch to **Saved Snippets** tab for card-based browsing with edit / copy / delete actions per snippet.

### 5. Generate a Markdown Report
Click **Report**, select an account from the dropdown, then click **Download Report**. A `.md` file is generated containing:
- Account summary (name, description, total count, category breakdown)
- Snippets grouped by category with title, tags, date, and body in fenced code blocks

### 6. Backup & Restore
Use **Export JSON** for full vault backup or **Import JSON** to merge snippets from an exported file (duplicates are skipped).

## API Reference

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/accounts` | List all accounts |
| `POST` | `/api/accounts` | Create a new account (`name`, `description`) |
| `PUT` | `/api/accounts/<id>` | Update an existing account |
| `DELETE` | `/api/accounts/<id>` | Delete an account (refused if snippets exist) |
| `GET` | `/api/snippets?q=...&account_id=...` | List snippets, with optional search query and/or account filter |
| `POST` | `/api/snippets` | Create a new snippet (`title`, `category`, `body`, `tags`, `account_id`) |
| `PUT` | `/api/snippets/<id>` | Update an existing snippet |
| `DELETE` | `/api/snippets/<id>` | Delete a single snippet |
| `POST` | `/review` | Send body text for AI grammar & content review (JSON `{body}`) |
| `POST` | `/api/extract` | Upload file â†’ AI extracts structured snippets (`multipart/form-data`) |
| `POST` | `/api/extract/save-all` | Bulk-save extracted snippets to an account |
| `GET` | `/api/export` | Download full JSON backup of all snippets |
| `POST` | `/api/import` | Upload a `.json` file to merge snippets into the vault |
| `GET` | `/api/export/markdown?account_id=<id>` | Download per-account Markdown report (file download) |

## Development Notes

- The application runs as a single Flask app with no frontend build step. Tailwind CSS is loaded via CDN; edit styles directly in `<style>` blocks or by swapping to a compiled asset pipeline if preferred.
- SQLite is used for simplicity and portability. For production deployments, swap `SQLALCHEMY_DATABASE_URI` to PostgreSQL or MySQL.
- LMStudio prompts are defined as module constants (`_EXTRACT_SYSTEM_PROMPT`, `_REVIEW_SYSTEM Prompt`) in `app.py` and can be tuned per team's knowledge-transfer style.
- Schema migrations are handled automatically on startup via `_migrate_schema()`.
